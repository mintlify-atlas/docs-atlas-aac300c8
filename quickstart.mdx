This guide walks you through running your first pgwd check, from a simple dry-run to a fully configured alerting setup.

## Prerequisites

- pgwd installed ([Installation guide](/installation))
- PostgreSQL database URL with credentials
- (Optional) Slack webhook or Loki endpoint for alerts

## Step 1: Install pgwd

If you haven't already, install pgwd:

<CodeGroup>
  ```bash Go
  go install github.com/hrodrig/pgwd@latest
  ```
  
  ```bash Homebrew
  brew install hrodrig/pgwd/pgwd
  ```
  
  ```bash Docker
  docker pull ghcr.io/hrodrig/pgwd:latest
  ```
</CodeGroup>

Verify the installation:

```bash
pgwd --version
```

## Step 2: Set your database URL

Export your PostgreSQL connection URL as an environment variable:

```bash
export PGWD_DB_URL="postgres://user:password@localhost:5432/mydb"
```

<Note>
  The database user needs read access to `pg_stat_activity` and `pg_settings`. Most standard roles have this by default.
</Note>

### URL format examples

<CodeGroup>
  ```bash Local Postgres
  postgres://postgres:secret@localhost:5432/postgres
  ```
  
  ```bash Remote with SSL
  postgres://user:pass@prod-host:5432/mydb?sslmode=require
  ```
  
  ```bash Disable SSL (development)
  postgres://postgres:secret@127.0.0.1:5433/postgres?sslmode=disable
  ```
  
  ```bash Kubernetes (after port-forward)
  postgres://postgres:DISCOVER_MY_PASSWORD@localhost:5432/mydb
  ```
</CodeGroup>

## Step 3: Run a dry-run check

A dry-run prints connection stats without sending any alerts:

```bash
pgwd -dry-run
```

You'll see output like:

```
total=42 active=3 idle=39 max_connections=100
```

This confirms pgwd can connect and read your database stats.

<Tip>
  Use `-dry-run` to test connectivity and see current connection counts before configuring thresholds.
</Tip>

## Step 4: Configure a notifier

Choose **Slack** or **Loki** (or both) for alerts.

<Tabs>
  <Tab title="Slack">
    ### Create a Slack webhook
    
    1. Go to [api.slack.com/messaging/webhooks](https://api.slack.com/messaging/webhooks)
    2. Create an Incoming Webhook for your workspace
    3. Copy the webhook URL
    
    ### Configure pgwd
    
    ```bash
    export PGWD_SLACK_WEBHOOK="https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXX"
    ```
    
    ### Test notification
    
    Send a test message to verify delivery:
    
    ```bash
    pgwd -force-notification
    ```
    
    You should receive a test message in your Slack channel.
  </Tab>
  
  <Tab title="Loki">
    ### Configure Loki endpoint
    
    Set your Loki push API URL:
    
    ```bash
    export PGWD_LOKI_URL="http://localhost:3100/loki/api/v1/push"
    ```
    
    ### Add labels (optional)
    
    Add custom labels for filtering in Grafana:
    
    ```bash
    export PGWD_LOKI_LABELS="app=pgwd,env=prod,db=myapp"
    ```
    
    ### Test notification
    
    Send a test log entry:
    
    ```bash
    pgwd -force-notification
    ```
    
    Query in Grafana:
    ```logql
    {app="pgwd", env="prod"}
    ```
  </Tab>
  
  <Tab title="Both">
    ### Configure both notifiers
    
    pgwd can send alerts to Slack and Loki simultaneously:
    
    ```bash
    export PGWD_SLACK_WEBHOOK="https://hooks.slack.com/services/..."
    export PGWD_LOKI_URL="http://localhost:3100/loki/api/v1/push"
    export PGWD_LOKI_LABELS="app=pgwd,env=prod"
    ```
    
    ### Test both
    
    ```bash
    pgwd -force-notification
    ```
    
    Check both Slack and Loki for the test message.
  </Tab>
</Tabs>

## Step 5: Run with thresholds

Now run a real check with thresholds.

### Option A: 3-tier percentage mode (default)

By default, pgwd uses 75/85/95% of `max_connections` as attention/alert/danger levels:

```bash
pgwd
```

This is equivalent to:

```bash
pgwd -threshold-levels 75,85,95
```

**How it works:**
- At 75% → **Attention** (yellow in Slack)
- At 85% → **Alert** (orange in Slack)
- At 95% → **Danger** (red in Slack)

Only the highest breached level triggers an alert.

### Option B: Explicit thresholds

Set specific connection counts:

```bash
pgwd -threshold-total 80
```

This alerts when total connections reach 80 or more.

### Option C: Multiple thresholds

Monitor multiple metrics in one run:

```bash
pgwd -threshold-total 90 -threshold-idle 60 -threshold-active 50
```

Each exceeded threshold generates a separate alert.

## Real-world examples

### One-shot: Alert on total connections

```bash
pgwd -db-url "postgres://user:pass@localhost:5432/mydb" \
     -threshold-total 80 \
     -slack-webhook "https://hooks.slack.com/services/..."
```

### Daemon: Check every 60 seconds

```bash
pgwd -db-url "postgres://user:pass@localhost:5432/mydb" \
     -threshold-total 80 \
     -interval 60 \
     -slack-webhook "https://hooks.slack.com/services/..."
```

Press Ctrl+C or send SIGTERM to stop gracefully.

### Detect stale connections (connection leaks)

Alert when any connection stays open longer than 10 minutes:

```bash
pgwd -db-url "postgres://user:pass@localhost:5432/mydb" \
     -stale-age 600 \
     -threshold-stale 1 \
     -slack-webhook "https://hooks.slack.com/services/..."
```

### Custom 3-tier levels

Use 70/85/90% instead of the default 75/85/95%:

```bash
pgwd -db-url "postgres://user:pass@localhost:5432/mydb" \
     -threshold-levels 70,85,90 \
     -slack-webhook "https://hooks.slack.com/services/..."
```

### Kubernetes: Connect via port-forward

Connect to Postgres in Kubernetes without manual port-forwarding:

```bash
pgwd -kube-postgres default/svc/postgres \
     -db-url "postgres://postgres:DISCOVER_MY_PASSWORD@localhost:5432/mydb" \
     -slack-webhook "https://hooks.slack.com/services/..."
```

pgwd runs `kubectl port-forward` automatically and discovers the password from the pod's environment (`POSTGRES_PASSWORD` by default).

<Note>
  Requires `kubectl` in your `PATH`. pgwd checks at startup and exits with a clear error if `kubectl` is missing.
</Note>

## Troubleshooting

### "missing database URL"

Set `PGWD_DB_URL` or use `-db-url`:

```bash
export PGWD_DB_URL="postgres://user:pass@localhost:5432/mydb"
```

### "no notifier configured"

Set at least one notifier or use `-dry-run`:

```bash
export PGWD_SLACK_WEBHOOK="https://hooks.slack.com/services/..."
```

### "postgres connect: connection refused"

Check that:
- Postgres is running and reachable
- Host, port, and database name are correct
- Credentials are valid
- Network/firewall allows connections

For local development with Docker:

```bash
# Start Postgres on host port 5433 to avoid conflicts
docker run -d --name pgwd-pg \
  -e POSTGRES_PASSWORD=secret \
  -e POSTGRES_DB=postgres \
  -p 5433:5432 \
  postgres:16-alpine

# Connect with sslmode=disable
export PGWD_DB_URL="postgres://postgres:secret@127.0.0.1:5433/postgres?sslmode=disable"
pgwd -dry-run
```

### No alerts received

Run a test notification to validate delivery:

```bash
pgwd -force-notification
```

Check:
- Webhook URL is correct
- pgwd can reach Slack/Loki (network/firewall)
- Slack channel or Loki labels are configured

### "kubectl not found in PATH"

Install kubectl and ensure it's in your PATH:

```bash
which kubectl
```

When running from cron, set PATH explicitly:

```bash
# In crontab
PATH=/usr/local/bin:/usr/bin:/bin
*/5 * * * * /usr/local/bin/pgwd ...
```
