---
title: Cron Deployment
description: Run pgwd on a schedule using cron for periodic one-shot checks
---

pgwd can run from cron for scheduled one-shot checks. This is ideal for periodic monitoring without a long-running daemon.

## Overview

Cron runs pgwd once per schedule tick. Each run:
1. Connects to PostgreSQL
2. Checks connection counts
3. Sends alerts if thresholds are exceeded
4. Exits

For one-shot mode, omit `PGWD_INTERVAL` or set it to `0`.

## Basic Cron Setup

<Steps>
  <Step title="Install pgwd">
    ```bash
    # From source
    go install github.com/hrodrig/pgwd@latest
    sudo cp $GOBIN/pgwd /usr/local/bin/pgwd

    # Or download from releases
    # wget https://github.com/hrodrig/pgwd/releases/download/v0.3.6/pgwd-linux-amd64.tar.gz
    # tar xzf pgwd-linux-amd64.tar.gz
    # sudo cp pgwd /usr/local/bin/pgwd
    ```
  </Step>

  <Step title="Test pgwd manually">
    Verify pgwd works before adding to cron:

    ```bash
    export PGWD_DB_URL="postgres://user:pass@localhost:5432/mydb"
    export PGWD_THRESHOLD_TOTAL=80
    export PGWD_SLACK_WEBHOOK="https://hooks.slack.com/services/..."

    /usr/local/bin/pgwd -dry-run
    ```
  </Step>

  <Step title="Add to crontab">
    ```bash
    crontab -e
    ```

    Add a line to check every 5 minutes:

    ```cron
    # Check pgwd every 5 minutes
    */5 * * * * PGWD_DB_URL="postgres://user:pass@localhost:5432/mydb" PGWD_THRESHOLD_TOTAL=80 PGWD_SLACK_WEBHOOK="https://hooks.slack.com/services/..." /usr/local/bin/pgwd >> /var/log/pgwd.log 2>&1
    ```
  </Step>
</Steps>

## PATH Requirements for kubectl

Cron runs with a **minimal environment** (`PATH=/usr/bin:/bin`). If you use `-kube-postgres`, cron must find `kubectl` in PATH.

<Warning>
Without proper PATH setup, pgwd will exit with "kubectl not found in PATH".
</Warning>

### Set PATH in crontab

```cron
# Set PATH before commands
PATH=/usr/local/bin:/usr/bin:/bin

# Run pgwd with kubectl access
*/5 * * * * /usr/local/bin/pgwd -kube-postgres default/svc/postgres -db-url "postgres://..." -slack-webhook "https://..." >> /var/log/pgwd.log 2>&1
```

### Use a wrapper script

Create a wrapper that sets PATH and runs pgwd:

<Steps>
  <Step title="Create wrapper script">
    ```bash
    sudo tee /usr/local/bin/pgwd-wrapper.sh > /dev/null << 'EOF'
#!/bin/sh
export PATH="/usr/local/bin:$PATH"
exec /usr/local/bin/pgwd "$@"
EOF

    sudo chmod +x /usr/local/bin/pgwd-wrapper.sh
    ```
  </Step>

  <Step title="Use wrapper in crontab">
    ```cron
    */5 * * * * /usr/local/bin/pgwd-wrapper.sh -kube-postgres default/svc/postgres -db-url "postgres://..." -slack-webhook "https://..." >> /var/log/pgwd.log 2>&1
    ```
  </Step>
</Steps>

## Environment Variables in Crontab

You can set environment variables directly in crontab:

```cron
# Environment variables
PGWD_DB_URL=postgres://user:pass@localhost:5432/mydb
PGWD_THRESHOLD_TOTAL=80
PGWD_THRESHOLD_IDLE=50
PGWD_SLACK_WEBHOOK=https://hooks.slack.com/services/...

# Run every 5 minutes
*/5 * * * * /usr/local/bin/pgwd >> /var/log/pgwd.log 2>&1
```

<Note>
Environment variables set in crontab apply to all commands below them. This keeps your cron lines clean.
</Note>

## Redirect Output to Log Files

Capture pgwd output for debugging:

```cron
# Append stdout and stderr to log file
*/5 * * * * /usr/local/bin/pgwd -db-url "postgres://..." -slack-webhook "https://..." >> /var/log/pgwd.log 2>&1
```

- `>>` appends stdout to the file
- `2>&1` redirects stderr to stdout (so errors go to the same log)

### Log rotation

Use logrotate to prevent log files from growing indefinitely:

```bash
sudo tee /etc/logrotate.d/pgwd > /dev/null << 'EOF'
/var/log/pgwd.log {
    daily
    rotate 7
    compress
    missingok
    notifempty
    create 0644 root root
}
EOF
```

## Multi-Service Monitoring Example

Monitor multiple PostgreSQL instances from a single script:

<Steps>
  <Step title="Create check script">
    Create `~/bin/pgwd-cron.sh`:

    ```bash
    #!/bin/bash
    mkdir -p ~/log
    export KUBECONFIG=/path/to/your/kubeconfig
    export PGWD_SLACK_WEBHOOK="https://hooks.slack.com/services/..."
    export PATH="/usr/local/bin:$PATH"
    PGWD=${PGWD:-/usr/local/bin/pgwd}

    echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) checking postgres-a"
    $PGWD -kube-postgres mynamespace/svc/postgres-a \
      -kube-local-port 15432 \
      -db-url 'postgres://postgres:DISCOVER_MY_PASSWORD@postgres-a:15432/db_a'

    echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) checking postgres-b"
    $PGWD -kube-postgres mynamespace/svc/postgres-b \
      -kube-local-port 15433 \
      -db-url 'postgres://postgres:DISCOVER_MY_PASSWORD@postgres-b:15433/db_b'

    echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) checking postgres-c"
    $PGWD -kube-postgres mynamespace/svc/postgres-c \
      -kube-local-port 15434 \
      -db-url 'postgres://postgres:DISCOVER_MY_PASSWORD@postgres-c:15434/db_c'

    exit 0
    ```
  </Step>

  <Step title="Make script executable">
    ```bash
    chmod +x ~/bin/pgwd-cron.sh
    ```
  </Step>

  <Step title="Add to crontab">
    ```bash
    crontab -e
    ```

    Add:
    ```cron
    PATH=/usr/bin:/bin
    */5 * * * * /bin/bash -l -c '~/bin/pgwd-cron.sh >> ~/log/pgwd.log 2>&1'
    ```
  </Step>
</Steps>

<Note>
Use distinct `-kube-local-port` values (15432, 15433, 15434) so multiple port-forwards don't conflict.
</Note>

## Heartbeat Pattern with Force Notification

Send periodic test notifications to verify the monitoring pipeline is working:

<Steps>
  <Step title="Create heartbeat script">
    Create `~/bin/pgwd-heartbeat.sh`:

    ```bash
    #!/bin/bash
    mkdir -p ~/log
    export KUBECONFIG=/path/to/your/kubeconfig
    export PGWD_SLACK_WEBHOOK="https://hooks.slack.com/services/..."
    export PATH="/usr/local/bin:$PATH"
    PGWD=${PGWD:-/usr/local/bin/pgwd}

    $PGWD -kube-postgres mynamespace/svc/postgres-a \
      -kube-local-port 25432 \
      -db-url 'postgres://postgres:DISCOVER_MY_PASSWORD@postgres-a:25432/db_a' \
      -force-notification

    $PGWD -kube-postgres mynamespace/svc/postgres-b \
      -kube-local-port 25433 \
      -db-url 'postgres://postgres:DISCOVER_MY_PASSWORD@postgres-b:25433/db_b' \
      -force-notification

    exit 0
    ```
  </Step>

  <Step title="Make script executable">
    ```bash
    chmod +x ~/bin/pgwd-heartbeat.sh
    ```
  </Step>

  <Step title="Add to crontab">
    Run heartbeat every 2 hours:

    ```cron
    PATH=/usr/bin:/bin
    0 */2 * * * /bin/bash -l -c '~/bin/pgwd-heartbeat.sh >> ~/log/pgwd.log 2>&1'
    ```
  </Step>
</Steps>

<Note>
Use different local ports for heartbeat (25432, 25433) than for checks (15432, 15433) to avoid conflicts if both scripts run close together.
</Note>

## Complete Crontab Example

Combine regular checks and heartbeat:

```cron
# Set PATH for all commands
PATH=/usr/bin:/bin

# Check all services every 5 minutes
*/5 * * * * /bin/bash -l -c '~/bin/pgwd-cron.sh >> ~/log/pgwd.log 2>&1'

# Send heartbeat every 2 hours
0 */2 * * * /bin/bash -l -c '~/bin/pgwd-heartbeat.sh >> ~/log/pgwd.log 2>&1'
```

## Troubleshooting

### Cron job not running

**Check cron service is running:**
```bash
sudo systemctl status cron
# or on some systems:
sudo systemctl status crond
```

**Verify crontab syntax:**
```bash
crontab -l  # List current crontab
```

**Check mail for errors:**
```bash
mail  # Cron sends errors via mail
```

### kubectl not found

**Problem:** pgwd exits with "kubectl not found in PATH".

**Solution:** Set PATH in crontab or use wrapper script (see [PATH Requirements](#path-requirements-for-kubectl)).

**Verify kubectl location:**
```bash
which kubectl  # Shows full path, e.g. /usr/local/bin/kubectl
```

Add that directory to PATH in crontab:
```cron
PATH=/usr/local/bin:/usr/bin:/bin
```

### No output in log file

**Check log file path exists:**
```bash
mkdir -p /var/log  # or ~/log
touch /var/log/pgwd.log
chmod 644 /var/log/pgwd.log
```

**Verify user has write permissions:**
```bash
ls -la /var/log/pgwd.log
```

### Cron runs but pgwd fails

**Test command manually:**
```bash
# Run the exact command from crontab
PGWD_DB_URL="postgres://..." /usr/local/bin/pgwd -dry-run
```

**Check environment variables:**
```bash
# Debug script to see cron environment
* * * * * env > /tmp/cron-env.txt
```

Compare cron environment with your shell environment.

### Permission denied on log file

**Problem:** Cron cannot write to log file.

**Solution:**
```bash
# If running as your user
touch ~/log/pgwd.log
chmod 644 ~/log/pgwd.log

# If running as root
sudo touch /var/log/pgwd.log
sudo chmod 644 /var/log/pgwd.log
```

### Port forward conflicts

**Problem:** Multiple pgwd instances use the same local port.

**Solution:** Use distinct `-kube-local-port` values for each instance:

```bash
# Service A uses 15432
pgwd -kube-postgres ns/svc/postgres-a -kube-local-port 15432 ...

# Service B uses 15433
pgwd -kube-postgres ns/svc/postgres-b -kube-local-port 15433 ...

# Service C uses 15434
pgwd -kube-postgres ns/svc/postgres-c -kube-local-port 15434 ...
```

## Cron Schedule Examples

| Schedule | Cron Expression | Description |
|----------|----------------|-------------|
| Every 5 minutes | `*/5 * * * *` | Most common for monitoring |
| Every 10 minutes | `*/10 * * * *` | Less frequent checks |
| Every hour | `0 * * * *` | Light monitoring |
| Every 30 minutes | `*/30 * * * *` | Moderate frequency |
| Every 2 hours | `0 */2 * * *` | Heartbeat checks |
| Daily at 9 AM | `0 9 * * *` | Daily reports |
| Every weekday at 8 AM | `0 8 * * 1-5` | Business hours only |

## Best Practices

1. **Always redirect output** to a log file for debugging:
   ```cron
   */5 * * * * /usr/local/bin/pgwd ... >> /var/log/pgwd.log 2>&1
   ```

2. **Use absolute paths** for commands and log files:
   ```cron
   # Good
   */5 * * * * /usr/local/bin/pgwd ... >> /var/log/pgwd.log 2>&1

   # Bad (may not work)
   */5 * * * * pgwd ... >> pgwd.log 2>&1
   ```

3. **Set PATH explicitly** when using kubectl:
   ```cron
   PATH=/usr/local/bin:/usr/bin:/bin
   ```

4. **Test commands manually first** before adding to cron:
   ```bash
   # Run the exact command you'll put in cron
   /usr/local/bin/pgwd -db-url "postgres://..." -dry-run
   ```

5. **Use wrapper scripts** for complex setups (multiple services, environment setup).

6. **Implement log rotation** to prevent disk space issues.

7. **Add heartbeat checks** with `-force-notification` to verify the monitoring pipeline.

8. **Use distinct ports** when monitoring multiple Kubernetes services to avoid port conflicts.
